## 1. Le Problème : Un Marché Aveugle

### La réalité du terrain africain

| Statistique | Source | Impact |
|-------------|--------|--------|
| **60%** des sites africains chargent en plus de 4 secondes | Interne / PageSpeed Insights | 53% des visiteurs quittent avant chargement |
| **73%** des sites africains ont une vulnérabilité critique non détectée | Interne / OWASP scan | Risque de piratage = arrêt d'activité |
| **78%** du trafic web se fait sur mobile | StatCounter Africa | La plupart des sites ne sont pas optimisés mobile |
| 1 seconde de retard = **7% de conversions perdues** | Akamai / Google | Chaque seconde compte directement en revenu |

### Pourquoi les outils existants échouent en Afrique

- **Langue** : Lighthouse, GTmetrix, SEMrush sont 100% anglais. Un patron de PME ne comprend pas "Cumulative Layout Shift".
- **Paiement** : Abonnements en dollar/euro, carte bancaire obligatoire. Inaccessible au budget d'une PME ivoirienne.
- **Actionnabilité** : Un score de 42/100 ne dit pas "quoi faire, combien ça coûte, qui le fait".
- **Contexte local** : Aucun outil ne prend en compte la latence réseau africaine, les hébergeurs locaux, les CMS dominants (WordPress à 80%).

**Résultat** : des milliers de PME africaines perdent des clients chaque jour sans savoir pourquoi ni comment arrêter l'hémorragie.

## 7. La Stack Technique : Léger, Rapide, Scalable

### Architecture

| Couche | Technologie | Justification stratégique |
|--------|-------------|---------------------------|
| **Frontend** | React 19 + Vite + TailwindCSS | Bundle optimisé, PWA-ready, dark mode premium, animations Framer Motion |
| **Backend** | Supabase (BaaS) | Auth, PostgreSQL, Row Level Security, temps réel, coût quasi nul à faible volume |
| **APIs** | Serverless Vercel | Scan, contact, paiement, abonnement — scaling automatique |
| **Tests** | Playwright E2E | Couverture critique (auth, scan, contact, paiement) |
| **Monitoring** | Sentry + Microsoft Clarity + UptimeRobot | Production-grade dès le jour 1 |
| **Email** | Resend | Transactionnels : nurture, confirmation, rejet, rappels |
| **Cache** | Supabase + TTL 24h | Optimisation coûts API Google (25k req/jour gratuites) |

### Sécurité et conformité

- Validation Zod sur tous les endpoints
- Sanitization HTML via DOMPurify
- Rate limiting + CORS configuré
- Row Level Security (RLS) Supabase
- Error boundaries React
- Pas de données privées stockées — uniquement résultats d'audit

### Points forts techniques

- **Lazy loading + code splitting** : chunks séparés (framework, animation, PDF, DB)
- **Cache intelligent 24h** : évite les coûts API, accélère les rescans
- **Hooks personnalisés** : `useScans`, `useAuth`, `useLiveStats` — codebase propre et maintenable
- **PWA ready** : vite-plugin-pwa pour installation mobile
